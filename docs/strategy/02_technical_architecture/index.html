    Technical Architecture: Evalcraft Platform¶

    "Pytest for AI Agents" — Complete System Design¶

    1. Design Philosophy & Core Principles¶

    1.1 Architectural North Star¶

    Evalcraft merges three proven paradigms into a single system:

    Paradigm Inspiration What We Borrow
    Record/Replay VCR.py, Docker Cagent Cassette-based capture of LLM + tool I/O for deterministic replay
    Test Runner pytest Plugin hooks, fixtures, decorators, conftest auto-discovery, assertion framework
    Eval Platform Braintrust, DeepEval Dataset-driven scoring, LLM-as-judge, regression tracking, CI integration

    1.2 Design Principles¶

    1. Zero-config capture — One decorator or context manager to start recording. No boilerplate.
    2. Framework-agnostic core — Adapters for LangGraph, CrewAI, OpenAI Agents SDK, Claude Agent SDK; core is framework-neutral.
    3. Determinism by default — Replay mode blocks all external I/O. Divergence = explicit test failure.
    4. Pytest-native — Ships as a pytest plugin. conftest.py, fixtures, markers, parametrize all work.
    5. Incremental adoption — Start with capture/replay (free), graduate to evals (SaaS), then CI regression (SaaS).
    6. Separation of concerns — Capture format is an open spec; eval scoring is pluggable; storage is swappable.

    LangGraph Adapter¶

    from evalcraft.adapters.base import EvalcraftAdapter
from langgraph.graph import StateGraph
from langgraph.checkpoint import BaseCheckpointSaver

class LangGraphAdapter(EvalcraftAdapter):
    """Adapter for LangGraph agents.

    Hooks into LangGraph's checkpoint system and node callbacks
    to capture state transitions, LLM calls, and tool executions.
    """
    framework_name = "langgraph"

    def attach(self, graph: StateGraph) -> None:
        # Wrap each node's function with our interceptor
        for node_name, node_fn in graph.nodes.items():
            graph.nodes[node_name] = self._wrap_node(node_name, node_fn)

        # Hook into the checkpoint saver for state capture
        if graph.checkpointer:
            original_put = graph.checkpointer.put
            graph.checkpointer.put = self._wrap_checkpoint(original_put)

    def _wrap_node(self, name: str, fn):
        async def wrapper(state, config):
            self._recorder.emit(TraceEvent(
                type="node_enter",
                node_name=name,
                state_snapshot=state,
                timestamp=time.time(),
            ))
            result = await fn(state, config)
            self._recorder.emit(TraceEvent(
                type="node_exit",
                node_name=name,
                state_snapshot=result,
                timestamp=time.time(),
            ))
            return result
        return wrapper

    OpenAI Agents SDK Adapter¶

    from agents import RunHooks, AgentHooks, RunContextWrapper, Agent
from evalcraft.adapters.base import EvalcraftAdapter

class OpenAIAgentsAdapter(EvalcraftAdapter):
    """Adapter for OpenAI Agents SDK.

    Uses the built-in RunHooks system to intercept agent lifecycle events.
    """
    framework_name = "openai_agents"

    def attach(self, agent: Agent) -> None:
        self._hooks = _EvalcraftRunHooks(self._recorder)
        # OpenAI Agents SDK natively supports RunHooks
        agent.hooks = self._hooks

class _EvalcraftRunHooks(RunHooks):
    def __init__(self, recorder):
        self._recorder = recorder

    async def on_agent_start(self, context: RunContextWrapper, agent: Agent):
        self._recorder.emit(TraceEvent(
            type="agent_start",
            agent_name=agent.name,
            model=agent.model,
            instructions=agent.instructions,
        ))

    async def on_tool_start(self, context, agent, tool):
        self._recorder.emit(TraceEvent(
            type="tool_start",
            tool_name=tool.name,
            tool_input=tool.input,
        ))

    async def on_tool_end(self, context, agent, tool, result):
        self._recorder.emit(TraceEvent(
            type="tool_end",
            tool_name=tool.name,
            tool_output=result,
        ))

    async def on_llm_end(self, context, agent, response):
        self._recorder.emit(TraceEvent(
            type="llm_response",
            model=agent.model,
            response=response,
            usage=response.usage,
        ))

    Claude Agent SDK Adapter¶

    from claude_agent_sdk import Agent, HookMatcher
from claude_agent_sdk.hooks import (
    PreToolUseHookInput, PostToolUseHookInput, StopHookInput
)
from evalcraft.adapters.base import EvalcraftAdapter

class ClaudeAgentAdapter(EvalcraftAdapter):
    """Adapter for Claude Agent SDK.

    Uses Claude's hook system (PreToolUse, PostToolUse, Stop, etc.)
    to capture full agent execution traces.
    """
    framework_name = "claude_agents"

    def attach(self, agent_options: dict) -> dict:
        """Inject Evalcraft hooks into Claude agent options."""
        hooks = agent_options.get("hooks", [])

        # Pre-tool-use: capture tool inputs
        hooks.append(HookMatcher(
            matcher=None,  # Match all tools
            callbacks=[self._pre_tool_hook],
        ))

        # Post-tool-use: capture tool outputs
        hooks.append(HookMatcher(
            matcher=None,
            callbacks=[self._post_tool_hook],
        ))

        agent_options["hooks"] = hooks
        return agent_options

    async def _pre_tool_hook(self, input: PreToolUseHookInput):
        self._recorder.emit(TraceEvent(
            type="tool_start",
            tool_name=input.tool_name,
            tool_input=input.tool_input,
        ))
        return {"decision": "allow"}

    async def _post_tool_hook(self, input: PostToolUseHookInput):
        self._recorder.emit(TraceEvent(
            type="tool_end",
            tool_name=input.tool_name,
            tool_output=input.tool_output,
        ))
        return {"decision": "allow"}

    Generic HTTP Proxy Adapter (Catch-All)¶

    class HTTPProxyAdapter(EvalcraftAdapter):
    """Framework-agnostic adapter using HTTP-level interception.

    Works with ANY agent framework by proxying all HTTP traffic
    to LLM providers (OpenAI, Anthropic, Google, etc.) through
    a local recording proxy. Similar to Docker Cagent's approach.

    Usage:
        # Set env vars to redirect traffic through proxy
        OPENAI_BASE_URL=http://localhost:9147/v1
        ANTHROPIC_BASE_URL=http://localhost:9147/v1
    """
    framework_name = "generic_http"

    def __init__(self, recorder, port: int = 9147):
        self._recorder = recorder
        self._port = port
        self._proxy = None

    async def start_proxy(self):
        """Start the local recording proxy server."""
        self._proxy = RecordingProxy(
            port=self._port,
            recorder=self._recorder,
            upstream_providers={
                "openai": "https://api.openai.com",
                "anthropic": "https://api.anthropic.com",
                "google": "https://generativelanguage.googleapis.com",
            },
        )
        await self._proxy.start()

    async def stop_proxy(self):
        await self._proxy.stop()

    """Result from a single scorer evaluation."""
    score: float              # 0.0 to 1.0
    passed: bool              # Binary pass/fail
    scorer_name: str          # Identifier
    explanation: Optional[str] = None  # Human-readable reason
    metadata: dict = None     # Additional scorer-specific data
    latency_ms: float = 0.0  # Time taken to evaluate

class Scorer(ABC):
    """Base class for all scorers."""

    name: str
    threshold: float = 0.5   # Default pass threshold

    @abstractmethod
    async def score(
        self,
        input: str,
        output: str,
        expected: Optional[str] = None,
        trace: Optional[AgentTrace] = None,
        context: Optional[dict] = None,
    ) -> EvalResult:
        ...

    6.4 ClickHouse Analytics Schema¶

    -- Time-series eval metrics for dashboards and trend analysis

CREATE TABLE eval_metrics
(
    -- Dimensions
    project_id      UUID,
    eval_run_id     UUID,
    golden_case_id  UUID,
    scorer_name     LowCardinality(String),
    git_commit      String,
    git_branch      LowCardinality(String),
    agent_model     LowCardinality(String),
    tags            Array(LowCardinality(String)),
    triggered_by    LowCardinality(String),

    -- Metrics
    score           Float32,
    passed          UInt8,
    tokens_used     UInt32,
    cost_usd        Float32,
    latency_ms      UInt32,
    llm_calls       UInt16,
    tool_calls      UInt16,

    -- Time
    timestamp       DateTime64(3),

    -- Org isolation
    org_id          UUID
)
ENGINE = MergeTree()
PARTITION BY toYYYYMM(timestamp)
ORDER BY (org_id, project_id, scorer_name, timestamp)
TTL timestamp + INTERVAL 2 YEAR;

-- Materialized view for score trends
CREATE MATERIALIZED VIEW eval_score_trends
ENGINE = AggregatingMergeTree()
PARTITION BY toYYYYMM(timestamp)
ORDER BY (org_id, project_id, scorer_name, day)
AS SELECT
    org_id,
    project_id,
    scorer_name,
    toDate(timestamp) AS day,
    avgState(score) AS avg_score,
    quantileState(0.5)(score) AS p50_score,
    quantileState(0.05)(score) AS p5_score,
    quantileState(0.95)(score) AS p95_score,
    countState() AS eval_count,
    sumState(tokens_used) AS total_tokens,
    sumState(cost_usd) AS total_cost
FROM eval_metrics
GROUP BY org_id, project_id, scorer_name, day;

    8.2 How It Works¶

    1. Bootstrap: Teams start with LLM-as-judge (expensive, accurate, ~$0.01/eval)
    2. Label collection: LLM judge scores + human feedback build training data
    3. Train classifier: Fine-tune DistilBERT on labeled data (F1 > 0.90 target)
    4. Deploy: Export to ONNX, serve via ONNX Runtime (~5ms/eval, ~$0.00001/eval)
    5. Monitor: Track classifier vs. LLM judge agreement; retrain when drift > 5%
    6. Fallback: If classifier confidence < threshold, route to LLM judge

from evalcraft.cloud.security import CassetteEncryptor

class CassetteEncryptor:
    """Encrypts cassettes before uploading to S3."""

    def __init__(self, org_key: bytes):
        self.org_key = org_key  # Derived from org's master key in KMS

    def encrypt(self, cassette_bytes: bytes) -> bytes:
        # AES-256-GCM with random nonce
        nonce = os.urandom(12)
        cipher = AES.new(self.org_key, AES.MODE_GCM, nonce=nonce)
        ciphertext, tag = cipher.encrypt_and_digest(cassette_bytes)
        return nonce + tag + ciphertext

    def decrypt(self, encrypted: bytes) -> bytes:
        nonce = encrypted[:12]
        tag = encrypted[12:28]
        ciphertext = encrypted[28:]
        cipher = AES.new(self.org_key, AES.MODE_GCM, nonce=nonce)
        return cipher.decrypt_and_verify(ciphertext, tag)

    11. Regression Detection Algorithm¶

    class RegressionDetector:
    """Detects regressions between eval runs."""

    async def detect(
        self,
        baseline: EvalRun,
        current: EvalRun,
        thresholds: RegressionThresholds,
    ) -> list[Regression]:
        regressions = []

        # 1. Overall score drop
        for scorer_name in baseline.summary.scores:
            baseline_mean = baseline.summary.scores[scorer_name]["mean"]
            current_mean = current.summary.scores[scorer_name]["mean"]
            delta = current_mean - baseline_mean

            if delta < -thresholds.score_drop_threshold:  # e.g., -0.05
                regressions.append(Regression(
                    type="score_drop",
                    severity="critical" if delta < -0.15 else "warning",
                    scorer=scorer_name,
                    baseline_score=baseline_mean,
                    current_score=current_mean,
                    delta=delta,
                ))

        # 2. New failures (cases that passed before but fail now)
        baseline_results = {r.golden_case_id: r for r in baseline.results}
        for result in current.results:
            baseline_result = baseline_results.get(result.golden_case_id)
            if baseline_result and baseline_result.passed and not result.passed:
                regressions.append(Regression(
                    type="new_failure",
                    severity="critical",
                    case_id=result.golden_case_id,
                    baseline_score=baseline_result.overall_score,
                    current_score=result.overall_score,
                ))

        # 3. Tool usage changes
        for result in current.results:
            baseline_result = baseline_results.get(result.golden_case_id)
            if baseline_result:
                baseline_tools = set(baseline_result.tools_used)
                current_tools = set(result.tools_used)
                if baseline_tools != current_tools:
                    regressions.append(Regression(
                        type="tool_change",
                        severity="warning",
                        case_id=result.golden_case_id,
                        details={
                            "added": list(current_tools - baseline_tools),
                            "removed": list(baseline_tools - current_tools),
                        },
                    ))

        # 4. Cost increase
        cost_increase = (
            (current.summary.total_cost - baseline.summary.total_cost)
            / baseline.summary.total_cost
        )
        if cost_increase > thresholds.cost_increase_threshold:  # e.g., 0.20 (20%)
            regressions.append(Regression(
                type="cost_increase",
                severity="warning",
                details={
                    "baseline_cost": baseline.summary.total_cost,
                    "current_cost": current.summary.total_cost,
                    "increase_pct": cost_increase * 100,
                },
            ))

        return regressions

    Appendix B: Key Design Decisions Log¶

    1. Why YAML cassettes, not binary? — Human readability is critical for debugging agent behavior. Developers need to read, edit, and diff cassettes. Binary formats (Protobuf, MessagePack) available as opt-in for large cassettes.

    2. Why pytest plugin, not custom runner? — pytest has 80%+ market share in Python testing. Building on pytest means zero learning curve, existing CI/CD configs work, and the ecosystem (parametrize, fixtures, markers, plugins) is free.

    3. Why proxy-based generic adapter? — New agent frameworks appear monthly. We can't write adapters for all of them. The HTTP proxy approach (inspired by Docker Cagent) works with ANY framework that calls LLM APIs over HTTP, providing day-one support for unknown frameworks.

    4. Why ClickHouse for analytics? — Eval metrics are classic time-series data (high write, aggregate read). ClickHouse provides 10-100x better performance than Postgres for these queries at fraction of the cost. PostgreSQL handles relational data; ClickHouse handles analytics. Clear separation.

    5. Why cascade scoring (classifier → LLM)? — Pure LLM-as-judge is $0.01/eval. At 10K evals/day = $100/day = $3K/month. Trained classifier at $0.00001/eval + 10% LLM fallback = $3/month. The cascade makes continuous eval economically viable.

    6. Why pgvector over dedicated vector DB? — At our scale (< 10M vectors), pgvector performance is sufficient. Co-locating vectors with relational data avoids operational complexity and data synchronization issues. Can migrate to dedicated vector DB if needed at >100M vectors.
